# To-Do / Remaining Implementations

This document tracks the final pieces needed to fully realize the **Internet of Agents** MVP for the hackathon demo. Some parts of the system are currently using "mock" implementations to stub out functionality and need to be wired up to their actual implementations.

## 1. Implement Real Smart Contract Interactions in the Frontend
Currently, when a user posts a task in the dashboard, the frontend routes the transaction to a hardcoded string `0xEscrowContract`. We need this to invoke the actual `TaskEscrow` smart contract.

**Files to update:** 
* `packages/frontend/src/pages/Tasks.tsx`

**Tasks:**
- [ ] Read the deployed contract address generated by Hardhat (`packages/contracts/deploy.ts` should output the address; store it in the frontend's `.env.local` or a config file).
- [ ] Replace `to: '0xEscrowContract'` in `Tasks.tsx` with the actual `TaskEscrow` deployed address.
- [ ] Add the actual contract ABI to the frontend to ensure that `encodeFunctionData` or the equivalent Luffa SDK transaction parameters correctly construct the data payload for the `postTask` solidity function.

**Files to create:**

## 3. Flesh out `packages/agents/` (Standalone Agent Hub)
Currently, `packages/agents/` is an empty shell. In a real generalized "Internet of Agents" network, developers should be able to pick up this package, write their own business logic, and have it securely execute off-chain logic using the `TaskEscrow` funds.

**Files to create:**
* `packages/agents/src/AgentClient.ts`
* `packages/agents/src/index.ts`

**Tasks:**
- [ ] Create a base class `AgentClient` that interacts with the `AgentRegistry` and listens for `AgentHired` events on the Endless blockchain.
- [ ] Provide an example "Research Agent" instantiation in `packages/agents/...` showing how a third-party dev connects their own Gemini/OpenAI API key, listens to the network, executes the prompt, and submits the result webhook back to the Orchestrator/Escrow.
